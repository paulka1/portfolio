AWS

La technologie Amazon Web Services

Amazon Web Services (AWS) est la plateforme cloud de Amazon. Elle la plus complète et la plus largement adoptée au monde. Elle propose plus de 200 services complets issus de centres de données du monde entier. Des millions de clients (des startups les plus dynamiques au monde, de très grandes entreprises et des agences fédérales de premier plan) utilisent AWS pour réduire leurs coûts, gagner en agilité et innover plus rapidement.

Je m'intéresse à la technologie AWS depuis plusieurs mois et j'ai pour objectif de passer les certifications afin de me perfectionner. Pour ce faire j'ai suivi plusieurs formations en ligne et je me suis inscris à AWS Educate qui est le programme de AWS. Pour ce portfolio j'ai d'ailleurs hébergé le site avec AWS.

Mon choix de m'intéresser à AWS est parce que selon moi l'utilisation du cloud pour un développeur logiciel est indispensable. Aws à l'avantage d'être le numéro un et d'être par conséquent très valorisé en entreprise.

Explications

Pour héberger et faire fonctionner le site j'ai eu recours à 4 fonctionnalités AWS. Le bucket S3, le API Gateway, les Lambda et le SES. Ces fonctionnalités m'ont permis d'héberger un site web static serverless pour 0€.

Amazon Simple Storage Service (abrégé S3) est un service de stockage de données. Le bucket s3 permet de stocker de la data sans mettre en route un serveur. On peut y stocker tous types de données et notamment un site web static. Une fois le site stocké dans le bucket S3 le site est visible par un lien généré par le bucket. Le problème est qu'il n'est visible que par moi et le formulaire de contact ne fonctionne pas.

Pour le fonctionnement du formulaire de contact j'ai choisi de me servir du service lambda pour m'éviter de mettre en route un serveur Ec2 payant qui aurait fonctionné non stop pour une faible utilité.

Généralement il faut mettre en route un serveur et héberger le code qui gère et envoie les données du formulaire de contact sur le serveur.

Une lambda permet de créer une fonction qui réagit à un déclencheur (trigger) comme le bouton envoyer du formulaire de contact sans à avoir à mettre en place un serveur qui tourne h24 en attente de cette évenement.

L'api Gateway est la porte qui fait le lien entre la lambda et le bucket S3. A la création de l'api gateway on peut choisir les méthodes souhaitées (POST, GET, ...) et l'api gateway génère une route api. Route utilisé pour liée le bucket s3 et le servir api gateway. L'api gateway déclenche ensuite la lambda.

Le service SES est le service Amazon Simple Email Service qui permet l'envoi d'email.

J'aurais pût ensuite utiliser le service S53 ou cloudFront pour modifier le nom de domaine et sécuriser. Pour passer de http://aws.us-east.portfolio.fr à https://paulcharendolPortfolio.com par exemple.
